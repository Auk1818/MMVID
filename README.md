# MMVID：基于 RGB 图像与毫米波雷达的跨模态行人重识别及三维人体重建系统

---

## 一、项目概述

随着智能监控与隐私保护需求的日益增长，如何在非接触、低侵扰的条件下实现精准的行人识别与人体姿态重建，已成为计算机视觉领域的一个重要研究方向。本项目正是在这一背景下展开的，我们提出并实现了一套完整的跨模态融合系统，该系统同时接收 **RGB 可见光图像** 与 **毫米波（mmWave）雷达点云** 作为输入，通过深度学习技术完成以下三项核心任务：

第一，**行人重识别（Person Re-identification）**，即在不同摄像头、不同时间场景下准确匹配同一行人的身份；第二，**三维人体姿态估计**，从二维图像和稀疏点云中恢复出人体的三维骨架结构与网格模型；第三，**生物特征推断**，包括身高和身体质量指数（BMI）的联合预测。

毫米波雷达的独特优势在于其能够穿透衣物、不受光照条件影响，这使其在隐私敏感场景（如更衣室、浴室周边监控）以及夜间或恶劣天气条件下具有不可替代的应用价值。然而，毫米波数据本身分辨率低、噪声大、语义信息稀疏，直接用于识别任务面临巨大挑战。因此，本项目的核心思路是利用 RGB 图像丰富的视觉语义来"引导"毫米波特征的学习，同时反过来用毫米波数据提供的几何结构信息来增强 RGB 特征的鲁棒性，形成一种双向互补的跨模态融合机制。

---

## 二、系统特性

本系统在设计过程中充分考虑了实际应用的需求和技术可行性，主要具备以下几方面的能力：

**多模态数据融合能力。** 系统能够同时处理 RGB 图像和毫米波点云两种异构数据，并通过专门设计的注意力机制在特征层面实现有效交互。RGB 分支利用卷积神经网络提取外观和纹理特征，而 mmWave 分支则采用 PointNet 系列网络处理不规则的三维点云数据。

**跨模态注意力交互。** 我们引入了非局部（Non-local）注意力模块，包含自注意力（Self-Attention）和互注意力（Mutual Attention）两个分支。前者负责建模各模态内部的时空依赖关系，后者则实现了 RGB 与 mmWave 之间的语义对齐和信息互补。

**完整的三维人体建模。** 系统集成了 SMPL（Skinned Multi-Person Linear）参数化人体模型，能够输出包含 6890 个顶点的精细人体网格以及 24 个关键关节点的三维坐标。这使得系统不仅能够回答"这是谁"，还能回答"这个人长什么样"、"他在做什么动作"。

**生物特征联合预测。** 在身份识别和姿态重建的基础上，我们进一步增加了身高和 BMI 的回归预测头。这些生物特征作为辅助监督信号，不仅丰富了模型的学习目标，还有助于提升主任务的泛化性能。

**时序信息建模。** 考虑到输入数据为视频序列形式，我们在网络中嵌入了双向 LSTM 模块，对连续帧之间的时间依赖关系进行显式建模，从而捕捉人体的运动模式。

**空间锚点聚合机制。** 受 PointNet++ 的启发，我们在 mmWave 分支中设计了基于预定义空间锚点的局部特征聚合策略。具体而言，三维空间被划分为规则的锚点网格，每个锚点聚合其邻域内的点云特征，再通过体素卷积进行进一步编码。

## 三、代码结构

下面从整体上介绍一下项目的文件组织方式。整个工程以 Python 语言编写，基于 PyTorch 深度学习框架构建，根目录下的文件组织如下：

**核心模型与训练代码。** `network.py` 是整个系统的核心，定义了所有神经网络模块的结构，包括点云处理分支、图像处理分支、注意力融合模块以及最终的预测头等。`train.py` 是主要的训练入口脚本，包含了完整的数据加载、模型训练循环、损失计算以及评估逻辑。此外还有一个较早版本的训练脚本 `train_ruili_pose_ruili_net_duanwu.py`，不包含生物特征损失部分，可用于对比实验。

**数据处理模块。** `dataset_me_rgb3.py` 中定义了 `Sensys_SMPL` 数据集类，负责从磁盘加载原始数据、执行训练/测试划分、以及动态构造三元组样本（锚点、正样本、负样本）。`losses.py` 实现了度量学习所需的各类损失函数，包括三元组损失（Triplet Loss）、对比损失（Contrastive Loss）及其在线采样变体。`metadata.py` 是一个辅助脚本，用于根据 34 位受试者的基本信息生成身高、体重和 BMI 的标注文件。`util.py` 则汇集了各种工具函数，涉及旋转矩阵运算、图像裁剪与变换、关键点绘制等功能。`ceshi.py` 是一个可视化脚本，可以在 RGB 图像上叠加绘制的骨骼连线，用于直观检验模型的预测效果。项目的依赖关系记录在 `requirements.txt` 文件中。

**第三方库引用（lib 目录）。** 该目录下存放了来自开源社区的基础代码，主要包括：SMPL 人体模型的 PyTorch 实现（`smpl.py`）、基于 SPIN 的姿态回归器（`spin.py`）、时序上下文建模模块（`tcmr.py`）、ResNet 骨干网络（`resnet.py`）以及运动判别器（`motion_discriminator.py`）。`pretrained/` 子目录保存了相关的预训练权重文件。此外还有几何计算（`geometry.py`）、渲染器（`renderer.py`）、可视化工具（`vis.py`）等实用模块。

**SMPL PyTorch 实现（smplpytorch 目录）。** 这是 SMPL 参数化人体模型的独立 PyTorch 封装，分为 `pytorch`（纯 PyTorch 实现）和 `native`（包含原始模型文件）两部分，提供了从前向传播到梯度计算的完整支持。

**图卷积网络模块（net 目录）。** 这里包含了空间-时间图卷积网络（ST-GCN）的实现，包括单流版本（`st_gcn.py`）和双流版本（`st_gcn_twostream.py`），以及对应的图结构定义（`graph.py`）和时序图卷积算子（`tgcn.py`）。这些模块为基于骨架数据的动作理解提供了基础架构。

**数据集目录（dataset 目录）。** `ruili34_nature/` 子目录存储了本次实验所使用的全部数据文件，均为 NumPy 的 `.npy` 格式。按照采集批次，数据被分为三组（后缀 `_1`、`_2`、`_3`），每组包含同一批受试者在不同时间段采集的数据。训练时会将三组数据合并后统一使用。具体的数据类型将在下一节详细说明。

**日志与模型输出目录。** `log/ruili_pose_duanwu/` 目录用于保存在训练过程中表现最优的模型权重文件（`model_best.pth`）。`loss/ruili_pose_duanwu/` 目录则保存了 TensorBoard 格式的训练日志，可以用于后续的可视化分析。

## 四、网络架构

### 4.1 总体设计思路

系统的网络架构遵循"双流编码—跨模态融合—多任务解码"的整体范式。下面用一个简化的流程图来说明数据在网络中的流动路径：

首先，RGB 图像序列经过一个基于 ResNet 的特征提取器（借鉴了 HMR/SPIN 的编码思路），得到每一帧的视觉特征向量；与此同时，毫米波点云序列经过 BasePointNet 进行逐帧的特征升维（从 3 维坐标扩展到 27 维特征），随后分别送入全局分支和锚点分支进行处理。全局分支通过 GlobalPointNet 提取带注意力加权的全局向量，再经双向 LSTM 编码时序信息，同时输出一个二维的全局位置偏移量——这一设计使得网络能够感知人体在场景中的大致位置。锚点分支则更为复杂：它先根据预设的空间锚点模板（覆盖人体周围的三维网格区域），利用 AnchorPointNet 聚合各锚点邻域内的点云特征，然后通过 AnchorVoxelNet（3D 卷积）将这些局部的锚点特征整合为体素级表示，最后同样由双向 LSTM 进行时序编码。

两条分支的输出——来自 RGB 视觉特征的查询向量（Query）和来自 mmWave 点云特征的键值向量（Key/Value）——一同送入跨模态非局部注意力模块。在该模块中，Self-Attention 分支确保每种模态能够充分挖掘自身内部的信息，而 Mutual-Attention 分支则让两种模态互相"参考"、互相增强。注意力模块的输出经过残差连接后，传入最终的融合预测头。

融合预测头（CombineModule_mid_modal_smpl）承担了所有的输出任务：它先将融合后的特征映射为 SMPL 模型的参数（包括 72 维的姿态参数、10 维的体型参数、3 维的相机参数以及 2 维的性别概率），然后调用 SMPL 层的前向传播函数生成 6890 个顶点的三维人体网格和 24 个关节点的坐标；与此同时，一条共享的特征提取分支从融合特征中提取出紧凑的生物特征表示，分别送入身高回归器和 BMI 回归器，得到两项生物指标的预测值。

### 4.2 各模块功能说明

为了让读者更好地理解每个组件的作用，下面逐一介绍网络中各个关键模块的设计意图和实现细节：

**BasePointNet（基础点云编码器）。** 这是最底层的点云处理单元，由三层一维卷积构成（通道数依次为 8→16→24），每层后接批量归一化和 ReLU 激活。它的输入是毫米波雷达采集到的原始点云坐标（每帧 64 个点，每个点 3 个坐标值），输出则是将原始 3 维坐标拼接上学习到的 24 维特征后的 27 维增强表示。这种"保留原始坐标+叠加学习特征"的设计有助于网络在后续处理中保持对空间位置的感知能力。

**GlobalPointNet（全局特征提取器）。** 在 BasePointNet 输出的基础上，GlobalPointNet 进一步通过三层卷积（32→48→64 通道）提取更高层的语义特征，并引入了一个可学习的注意力机制：对每一帧的所有点计算注意力权重，然后加权求和得到一个固定长度的全局特征向量。这种设计使得网络能够自适应地关注点云中对当前任务更重要的区域（例如人体躯干部分通常比背景散点更重要）。

**AnchorPointNet / AnchorVoxelNet（锚点特征提取器）。** 这一对模块共同实现了局部特征的空间化聚合。AnchorPointNet 的结构与 GlobalPointNet 类似，但它处理的不是全部的点，而是每个空间锚点周围最近邻的 K 个点（实验中 K=8）。AnchorVoxelNet 接收所有锚点的特征后，将其排列成三维体素网格的形式，然后用三层 3D 卷积（96→128→64 通道）进行空间上的信息传播和压缩，最终输出一个固定的向量表示。这种"先局部聚合、再体素卷积"的策略能够在保持计算效率的同时捕获点云中的局部几何结构。

**GlobalRNN / AnchorRNN（时序编码器）。** 两者均为三层双向 LSTM 结构，隐藏维度为 64，并配置了 0.1 的 Dropout 以防止过拟合。GlobalRNN 处理的是全局特征的时间序列，除了输出时序隐状态外，还通过一个额外的全连接层（64→16→2）预测每帧的二维位置偏移量，这一偏移量会反馈给 Anchor 模块用于动态调整锚点的空间位置，形成一种位置感知的自适应机制。AnchorRNN 则专注于编码锚点特征的时序演化规律。

**NonLocalBlock（非局部注意力块）。** 这是整个跨模态融合的核心。该模块同时维护 Self-Attention 和 Mutual Attention 两条信息通路。对于 Self-Attention，模态内的特征自身作为 Query、Key 和 Value 来计算注意力图，捕获长距离的空间和时间依赖；对于 Mutual Attention，一种模态的特征作为 Query，另一种模态的特征作为 Key 和 Value，实现跨模态的信息检索和对齐。两条通路的输出经过各自独立的变换和归一化后，再拼接在一起进行最终映射。值得注意的是，输出变换层的权重初始化为零，这在训练初期相当于恒等映射，有助于保持训练稳定性。

**CombineModule_mid_modal_smpl（多任务预测头）。** 这是网络的"终端"，负责将抽象的融合特征转化为具体的预测结果。SMPL 相关的部分包括四个全连接分支，分别预测姿态（72维）、体型（10维）、相机参数（3维）和性别（2维），然后将这些参数送入 SMPL 层得到顶点和关节点。生物特征部分采用了一个两阶段的共享结构：先用一个 256→128 的全连接层提取共享的生物表征，再分别接 128→1 的线性层预测身高和 BMI。这种共享-分离的设计既减少了参数量，又允许两项预测任务利用彼此学到的中间表示。

### 4.3 跨模态注意力的工作原理

跨模态注意力机制是本系统区别于传统单模态方法的关键所在。其工作流程可以概括为以下几个步骤：

第一步，特征嵌入。两种模态的特征分别通过独立的 1×1 卷积（针对图像特征）或 1×1 卷积（针对点云特征）投影到一个共同的嵌入空间中，生成 Query、Key、Value 三组表示。

第二步，注意力分数计算。Self-Attention 分支中，Query 和 Key 来自同一种模态，计算得到的注意力图反映了该模态内部不同空间位置之间的相关性；Mutual-Attention 分支中，Query 来自一种模态（例如 RGB），而 Key 来自另一种模态（例如 mmWave），计算得到的注意力图则刻画了跨模态的对应关系。

第三步，加权聚合与输出。根据注意力分数对 Value 进行加权求和，得到上下文增强后的特征表示。Self-Attention 的输出经过批量归一化和 ReLU 激活后，与原始输入相加形成残差连接；Mutual-Attention 同样如此。两者的输出拼接后再经过一次线性变换和归一化，得到最终的融合特征。

这种设计的优点在于：它不需要显式的模态对齐标注（如图像与点云之间的像素级对应关系），而是完全通过端到端的学习自动发现两种模态之间的关联模式。

## 五、数据集说明

### 5.1 数据来源与规模

本次实验使用的数据集是在实验室环境下自行采集的，我们称之为（ruili34_nature）。该数据集共包含 **34 名受试者**，每位受试者在不同的时间段完成了 **20 组** 动作序列的采集，因此总共有 **680 个样本**。每组序列包含连续的 **20 帧数据**，但在实际训练和推理中，我们只使用最后 **10 帧**（因为初始几帧可能包含较多的过渡动作或不稳定信号）。数据按照 7:3 的比例随机划分为训练集和测试集，划分时确保同一个人的样本不会同时出现在训练集和测试集中，以保证评估结果的可靠性。

### 5.2 数据类型详解

每位受试者在每次采集中，同步记录了以下多种模态的数据：

**毫米波雷达点云数据（list_all_ti）。** 这是本系统中较有特色的一种数据类型。毫米波雷达通过发射和接收电磁波来探测环境中物体的距离、角度和速度信息。在我们的设置中，每帧数据包含 64 个空间点的三维坐标（x, y, z），形状为 (N, 20, 64, 3)，其中 N 为样本总数（680），20 为帧数。这些点云大致勾勒出了人体的空间轮廓，但由于雷达分辨率的限制，点云通常比较稀疏且含有噪声。在实际加载时，我们只保留了前三个坐标分量，舍弃了可能存在的速度或反射强度信息。

**RGB 图像数据（list_all_image）。** 同时采集的可见光相机图像，分辨率为 224×224 像素，三通道彩色，形状为 (N, 20, 224, 224, 3)。这些图像提供了丰富的外观和纹理信息，是传统行人重识别方法的主要输入源。需要注意的是，由于原始数据的通道顺序与 PyTorch 的约定不一致，在加载时需要执行转置操作将通道维移到前面。

**Kinect 骨架关键点（list_all_kinect_key）。** 作为姿态估计的真值标签，我们使用 Microsoft Kinect 传感器同步采集了 24 个三维骨架关节点的坐标，形状为 (N, 20, 24, 3)。这些关键点涵盖了从头到脚的主要身体部位，包括头部、颈部、肩部、肘部、腕部、髋部、膝盖、踝部等。在计算关键点重建损失时，我们选取了其中 20 个主要关节点（排除了某些容易产生噪声的末端点）进行误差衡量。

**SMPL 模型参数（list_all_smpl_vert / list_all_smpl_key）。** 除了 Kinect 关键点外，我们还通过拟合 SMPL 模型得到了更精细的人体表示。SMPL 顶点数据包含 6890 个三维顶点坐标 (N, 20, 6890, 3)，构成了完整的人体表面网格；SMPL 关节数据则包含 24 个关节点的坐标 (N, 20, 24, 3)，这些关节点是 SMPL 模型的内置骨架，与 Kinect 关节点在定义上有所不同但语义对应。

**身份标签（list_label_all）。** 每个样本对应的行人 ID，取值范围为 0 到 33（共 34 个类别），形状为 (N,)。这是行人重识别任务的核心监督信号。

**性别标签（list_all_gender）。** 每位受试者的性别信息，用于 SMPL 模型选择合适的身体模板（男性/女性）。

**生物特征数据（list_all_height / list_all_bmi / list_all_weight）。** 包括身高（单位：厘米）、身体质量指数和体重（单位：千克）。这些数据来源于受试者填报的基本信息，每位受试者的 20 组序列共享相同的生物特征值（因为在一次数据采集 session 中人的身高和体重不会改变）。这部分数据由 `metadata.py` 脚本根据预先填写的表格生成并保存。

需要特别说明的是，上述所有数据按采集批次被分成了三个独立的文件组（后缀分别为 `_1`、`_2`、`_3`），每组大约包含 200 多个样本。这样做的原因是数据采集跨越了多个时间段，每次采集的环境配置可能略有不同。在训练时，程序会将三组数据合并后统一打乱和使用。

## 六、训练方法

### 6.1 环境准备

在开始训练之前，需要确保系统中已安装了正确版本的 Python 及相关依赖库。本项目推荐使用 **Python 3.8 或更高版本** 以及 **PyTorch 1.7.0 以上版本**（需支持 CUDA 以利用 GPU 加速）。完整的依赖列表记录在 `requirements.txt` 文件中，可以通过以下命令一键安装：


### 6.4 损失函数设计

本系统的总损失函数由多个子任务损失加权组合而成，其数学表达式为：

**L_total = a × L_kp + L_loc + a × L_metric + b × L_biometric**

下面分别解释每一项的含义和设计动机：

**L_kp — 关键点重建损失。** 这是一项监督学习损失，衡量模型预测的三维骨架关键点与真值（Kinect 采集的关键点）之间的差异。具体而言，我们选取了 20 个主要关节点（排除了一些噪声较大的末端点），计算预测坐标与真实坐标在每个点上的欧氏距离，然后对所有点和所有样本取均值，得到 MPJPE（Mean Per Joint Position Error，平均每关节位置误差）。为了促进各模态独立学习有效的姿态表示，我们还加入了一项单模态自监督损失，即在跨模态融合之前，要求每个模态单独输出的关键点预测也要尽量接近真值。最终的 L_kp 是融合后损失和单模态自监督损失的等权平均。

**L_loc — 位置损失。** 全局 LSTM 分支在编码时序特征的同时，还会输出一个二维的位置偏移预测（代表人体在图像平面上的大致中心位置）。我们将这个预测值与真值关键点的根关节（通常是髋部中心）的前两个坐标进行比较，计算 MSE 损失。这项损失的作用是为网络提供一个辅助的空间定位信号，帮助 Anchor 模块更准确地调整锚点的位置。

**L_metric — 度量学习损失。** 这是行人重识别任务的核心损失，采用 Triplet Loss（三元组损失）的形式。每次训练迭代中，模型会接收到一个三元组样本：锚点（Anchor）、正样本（Positive，与锚点属于同一人）和负样本（Negative，与锚点属于不同人）。Triplet Loss 要求锚点到正样本的特征距离小于锚点到负样本的特征距离，且差距至少为 margin（边界值）。在本系统中，我们设计了两组三元组约束：第一组的锚点是"RGB + 正样本 mmWave"的融合特征，正样本是"正样本 mmWave + 锚点 RGB"的融合特征，负样本是"负样本 mmWave + 负样本 RGB"的融合特征；第二组则交换了角色分配。这种交叉配对的设计迫使网络学习到模态不变的身份特征表示。

**L_biometric — 生物特征损失。** 这是一项均方误差（MSE）损失，同时对身高和 BMI 两项生物指标进行回归监督。对于三元组中的每一个样本（锚点、正样本、负样本），模型都会输出对应的身高和 BMI 预测值，然后与真实的标注值计算 MSE。六个预测值的 MSE 取平均后得到最终的 L_biometric。引入这项损失的初衷有两个方面：一是提供额外的监督信号，防止模型在有限的 ID 标签上过拟合；二是生物特征本身与人体外观存在一定的相关性，学习预测这些特征可能会帮助模型捕获更有判别力的中间表示。

### 6.5 评估指标

在训练过程中和测试阶段，我们使用以下指标来量化系统的性能：

**Rank-N 准确率。** 对于每一个查询样本（Query），系统将其特征与图库（Gallery）中所有样本的特征进行相似度排序，如果排在第 N 位及以前的样本中有与查询同ID的样本，则视为命中。Rank-1、Rank-5、Rank-10 分别表示在前 1、5、10 个结果中找到匹配的成功率。这是行人重识别领域最常用的评价指标。

**mAP（mean Average Precision）。** 对每个查询样本计算其 Precision-Recall 曲线下的面积（Average Precision），然后在所有查询样本上取平均值。mAP 综合考虑了排序位置和查准率，是对检索质量的更全面衡量。

**MPJPE。** 如前所述，这是三维关键点重建误差的度量单位为毫米，值越小表示姿态估计越精确。

## 七、模型接口说明

### 7.1 输入格式

模型的输入包含以下张量：

-   **data_rgb**：锚点样本的 RGB 图像序列，形状为 (B, T, C, H, W)，其中 B 为批次大小（默认 4），T 为序列长度（默认 10），C 为颜色通道数（3），H 和 W 为图像的高和宽（224）。
-   **data_rgb2**：负样本的 RGB 图像序列，形状与 data_rgb 相同。
-   **data_ti**：正样本的毫米波点云序列，形状为 (B, T, N, 3)，其中 N 为每帧点数（默认 64），3 表示 xyz 三个坐标。
-   **data_ti2**：负样本的毫米波点云序列，形状与 data_ti 相同。
-   **h0, c0**：双向 LSTM 的初始隐状态和细胞状态，形状均为 (层数×2, B, 隐藏维度)，即 (6, 4, 64)。

### 7.2 输出格式

模型返回的结果是一个多元组，包含以下内容：

-   **rgb_l, ti_l, ti_l2**：分别是三种组合方式（锚点RGB+正样本mmWave、正样本mmWave+锚点RGB、负样本mmWave+负RGB）融合后的 ReID 特征向量，用于计算度量学习损失和最终的相似度匹配。
-   **key_pre_rgb, key_pre_ti, key_pre_ti2**：上述三种组合方式融合后预测的 24 个三维关节点坐标，形状为 (B×T, 24, 3)，用于计算关键点重建损失。
-   **key_pre_rgb_self, key_pre_ti_self, key_pre_ti_self_n**：各模态在融合之前单独预测的关节点坐标，用于计算自监督损失。
-   **g_loc_p, g_loc_n**：正样本和负样本对应的全局二维位置偏移预测。
-   **output_rgb, output_ti, output_ti2**：三个字典，每个字典包含 SMPL 输出（vertices: 6890×3 的顶点坐标，joints: 24×3 的关节点坐标，cam: 3维相机参数，gender: 2维性别概率）以及生物特征预测（height: 标量身高，bmi: 标量 BMI）。

## 九、技术创新点总结

回顾整个系统的设计和实现过程，我们认为以下几个方面具有一定的创新性和实用价值：

**隐私保护与识别精度的平衡。** 传统的人脸识别或基于外表特征的行人重识别方法在隐私敏感场景中受到严格限制。毫米波雷达作为一种非光学传感器，天然具备隐私保护特性（无法从雷达数据中还原出人脸图像）。本系统证明了即使在这种"低质量"的传感模态下，通过与 RGB 图像的合理融合，仍然可以达到可用的识别精度，为隐私保护型监控系统提供了一种可行的技术路线。

**不对称跨模态融合框架。** 与大多数假设两种模态数据质量相当的多模态融合工作不同，本系统面对的是一种典型的不对称场景：RGB 信息丰富但易受环境影响（光照、遮挡），mmWave 信息稀疏但稳定可靠。我们的注意力机制设计充分考虑了这种不对称性，让信息丰富的模态"主导"融合过程，同时让稳定的模态提供"校准"信号。

**多任务协同学习的正则化效果。** 行人重识别本质上是一个分类/度量学习问题，当训练数据有限时（34 人 × 20 序列 = 680 样本），模型很容易过拟合。通过同时引入姿态重建和生物特征预测这两个辅助任务，相当于为模型增加了额外的监督信号和归纳偏置，实验表明这确实提升了主任务的泛化性能。

**空间感知的动态锚点机制。** 将静态的空间锚点与 LSTM 预测的位置偏移相结合，使锚点能够跟随人体的实际位置进行调整。这种设计灵感来自于目标检测领域的可变形卷积思想，但首次被应用到了点云处理和跨模态融合的场景中。
